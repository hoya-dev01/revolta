테스트 커버리지를 분석하고 부족한 테스트를 작성해줘: $ARGUMENTS

## 분석 절차

1. `pnpm test -- --coverage`를 실행하여 현재 커버리지 확인
2. 커버리지가 80% 미만인 파일을 식별
3. 누락된 테스트 케이스를 분류:
   - 핵심 로직 (비즈니스 로직, 데이터 변환)
   - 에러 핸들링 (실패 시나리오, 엣지 케이스)
   - 통합 (컴포넌트 간 상호작용)
4. 우선순위가 높은 순서대로 테스트 작성

## 테스트 작성 규칙

- Vitest의 `describe` / `it` 패턴 사용
- 테스트 설명은 한국어로 작성 가능 (예: `it('빈 대화는 무시해야 한다')`)
- 각 테스트는 하나의 동작만 검증
- Mock은 최소한으로 사용, 가능하면 실제 모듈 사용
- Content script 테스트는 DOM 환경 모킹 필수 (`jsdom`)

⚠️ 소스 코드(src/)는 수정하지 마. 테스트 파일(tests/)만 작성해.
