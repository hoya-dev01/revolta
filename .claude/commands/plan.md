다음 기능에 대한 구현 계획을 작성해줘: $ARGUMENTS

아래 형식으로 `docs/plans/` 디렉토리에 마크다운 파일로 저장해줘.
파일명은 GitHub 이슈 번호를 접두사로 사용해 (예: `#10` 이슈라면 `0010-feature-name.md`).

---

## 계획 형식

### 목표
(이 기능이 해결하는 문제와 기대 결과)

### 범위
(포함되는 것과 포함되지 않는 것)

### 구현 단계
- [ ] Step 1: ...
- [ ] Step 2: ...
(각 단계에 예상 난이도 표시: 🟢 쉬움 / 🟡 보통 / 🔴 어려움)

### 영향받는 파일
(생성/수정될 파일 목록)

### 멀티 에이전트 활용 계획
(에이전트 분리가 필요한 경우 역할 분담 명시)
- Agent 1 (codegen): ...
- Agent 2 (tester): ...
- Agent 3 (reviewer): ...

### 리스크 및 대응
(예상되는 문제와 대응 방안)

### 의존성
(선행되어야 하는 작업이나 외부 의존성)

---

⚠️ 계획만 작성하고, 코드는 절대 작성하지 마.
작성 후 사용자에게 계획을 보여주고 승인을 기다려.
