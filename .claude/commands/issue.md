GitHub 이슈를 읽고 사용자의 명령을 수행해줘.

인자: $ARGUMENTS

## 절차

1. `git remote get-url origin`으로 현재 리포지토리의 GitHub 원격 URL을 확인해.
2. $ARGUMENTS에서 이슈 번호를 추출해. 숫자만 있으면 그게 이슈 번호이고, "10번", "#10", "10번 이슈" 같은 패턴도 인식해.
3. `gh issue view <이슈번호> --repo <owner/repo>`로 해당 이슈의 전체 내용을 읽어.
4. 이슈 내용(제목, 본문, 작업 항목, 의존 관계 등)을 컨텍스트로 파악해.
5. $ARGUMENTS에서 이슈 번호를 제외한 나머지 부분이 사용자의 명령이야. 그 명령을 이슈 컨텍스트를 기반으로 수행해.
   - 명령이 없으면 이슈 내용을 요약해서 보여주고 "어떤 작업을 수행할까요?" 라고 물어봐.
   - 명령 예시: "수행해줘", "구현해줘" → 이슈의 작업 항목을 기반으로 구현 시작
   - 명령 예시: "분석해줘" → 이슈 내용과 현재 코드베이스 상태를 분석
   - 명령 예시: "계획 세워줘" → 이슈 기반으로 구현 계획 작성

## 주의사항

- 이슈를 읽은 후 반드시 이슈 제목과 핵심 내용을 사용자에게 간단히 보여줘.
- 이슈에 의존 관계가 있으면 선행 이슈가 완료되었는지 확인해.
- CLAUDE.md의 Workflow Rules(계획 우선 원칙 등)를 준수해.
- gh CLI가 인증되어 있지 않으면 사용자에게 안내해.
